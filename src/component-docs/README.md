# Component Docs (MDX)

This folder powers the internal Components Gallery. Docs are written in MDX, discovered automatically, linted for
consistency, and rendered in the docs UI.

## Quick start

Create a new doc:

```bash
yarn docs:new
```

Lint all docs:

```bash
yarn docs:lint
```

View in UI (local):

* [http://localhost:3000/med-legal/components](http://localhost:3000/med-legal/components)

## Folder structure

Categories are folder driven.

```txt
src/component-docs/
  ui/                 # UI primitives (Button, Dialog, etc.)
  shared/             # Shared/cross-cutting components
  forms/              # Form-related components
  code-block/         # Code display components
  docs-layout/        # Documentation layout components
  gallery/            # Gallery page internals
  registry/           # Registry logic
  _template/          # MDX template (not a category)
  scripts/            # CLI tools (not a category)
  docs/               # Internal docs (not a category)
```

Any folder under `src/component-docs/` becomes a category, except:

* folders starting with `_`
* `scripts`
* `docs`

## Frontmatter

Every MDX doc must include these required fields:

```md
---
id: toggle-group
title: Toggle Group
section: Components
category: UI
description: A group of toggleable buttons.
tags: ["ui", "toggle", "group"]
---
```

| Field         | Required | Notes                                        |
|---------------|----------|----------------------------------------------|
| `id`          | yes      | Kebab case, unique across all docs           |
| `title`       | yes      | Display name in the gallery                  |
| `section`     | yes      | Always `Components`                          |
| `category`    | yes      | Must match folder category after normalization (uppercase) |
| `description` | yes      | One sentence, used by registry and linter    |
| `tags`        | no       | String array, auto-generated by the CLI      |

## Key rules

* Frontmatter `id` is the stable identifier (must be kebab case and unique)
* Frontmatter `category` must match the folder category after normalization
* All five required frontmatter fields must be present and non-empty
* Do not put docs under `_template/`, `scripts/`, or `docs/`
* Run `yarn docs:lint` before committing any docs changes
