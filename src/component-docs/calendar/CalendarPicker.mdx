## Default — single mode (zero config)

The bare picker. TooltipButton with a calendar icon as the default trigger; clicking opens a single-mode Calendar; clicking a day fires `onChange(iso)` and auto-closes.

export const SingleDefaultDemo = () => {
  const [iso, setIso] = useState(null);
  return (
    <div className="flex items-center gap-3">
      <CalendarPicker value={iso} onChange={setIso} />

      <span className="text-muted text-sm">{iso ?? 'No date picked'}</span>
    </div>
  );
};

## Range mode — two-click range selection

`mode="range"` switches to two-click range selection. `onChange` fires only when BOTH `from` and `to` are picked; the popover stays open during partial selection and closes after the range completes.

export const RangeDemo = () => {
  const [range, setRange] = useState(null);
  return (
    <div className="flex items-center gap-3">
      <CalendarPicker
        mode="range"
        value={range}
        onChange={setRange}
        closeOn="never"
        triggerTooltip="Pick a date range"
      />

      <span className="text-muted text-sm">
        {range ? `${range.from} → ${range.to}` : 'No range picked'}
      </span>
    </div>
  );
};

## Content override — function-children with picker API

Render a custom calendar UI inside the popover. The function receives `selected`, `mode`, `onSelect`, and `close` — call `onSelect(value)` to fire `onChange`, then `close()` when ready.

export const ContentFunctionDemo = () => {

  return (

                    {date}

              </div>
            </div>
          )}
        </CalendarPicker.Content>
      </CalendarPicker>

      <span className="text-muted text-sm">{iso ?? 'No date picked'}</span>
    </div>

## Controlled open state

Manage popover open/close from the consumer side via `open` + `onOpenChange`. Useful when an external control needs to drive the picker, or when the consumer needs to suppress auto-close (combine with `closeOn="never"`).

The toggle button below is mounted as `<CalendarPicker.Trigger>` (with function-children to read `isOpen`) so Radix manages the click natively — clicking the button outside the trigger scope while the popover is open would race with Radix's outside-click handler and reopen the popover.

export const ControlledOpenDemo = () => {
  const [iso, setIso] = useState(null);
  const [open, setOpen] = useState(false);
  return (
    <div className="flex items-center gap-3">
      <CalendarPicker value={iso} onChange={setIso} open={open} onOpenChange={setOpen}>
        <CalendarPicker.Trigger>
          {({ isOpen }) => (
            <Button size="sm" variant="outline">
              {isOpen ? 'Close picker' : 'Open picker'}
            </Button>
          )}
        </CalendarPicker.Trigger>
      </CalendarPicker>

      <span className="text-muted text-sm">{iso ?? 'No date picked'}</span>
    </div>
  );
};

<ControlledOpenDemo />

## Non-dismissable popup (`dismissable={false}`)

Disables Radix's auto-dismiss behaviors — clicking outside the popover or pressing `Esc` no longer closes it. Only the trigger button (or any consumer-controlled action like a toggle / X button / Apply button) drives open/close. Pair with controlled `open` + `closeOn='never'` for full consumer control of the open/close lifecycle.

Use cases: date+time pickers requiring explicit Apply, multi-step UIs that mustn't lose state on outside-click, modal-style pickers where the popover stays open until the consumer explicitly dismisses it.

export const NonDismissableDemo = () => {
  const [iso, setIso] = useState(null);
  const [open, setOpen] = useState(false);
  return (
    <div className="flex items-center gap-3">
      <CalendarPicker
        value={iso}
        onChange={setIso}
        open={open}
        onOpenChange={setOpen}
        dismissable={false}
        closeOn="never"
      >
        <CalendarPicker.Trigger>
          {({ isOpen }) => (
            <Button size="sm" variant="outline">
              {isOpen ? 'Close picker' : 'Open picker'}
            </Button>
          )}
        </CalendarPicker.Trigger>
      </CalendarPicker>

      <span className="text-muted text-sm">{iso ?? 'No date picked'}</span>
    </div>
  );
};

<NonDismissableDemo />

Try clicking outside the popover or pressing `Esc` — the popover stays open. Only the toggle button dismisses it.

## Date + time picker — custom Content with deferred close

Picks a date AND a time before committing. Uses `closeOn="never"` + controlled `open` so the popover stays open while the user picks both, then an Apply button consolidates the value and closes via `close()` from the slot API.

export const DateTimePickerDemo = () => {
  const { toBoundaryDate, toDate } = useDate();
  const [dateTime, setDateTime] = useState(null);
  const [pendingDate, setPendingDate] = useState(null);
  const [pendingTime, setPendingTime] = useState('12:00');
  const [open, setOpen] = useState(false);
  return (
    <div className="flex items-center gap-3">
      <CalendarPicker
        value={pendingDate}
        onChange={setPendingDate}
        open={open}
        onOpenChange={setOpen}
        closeOn="never"
        triggerTooltip="Pick date + time"
      >
        <CalendarPicker.Content side="bottom" align="start">
          {({ close }) => (
            <div className="flex flex-col gap-3 p-3">
              <Calendar
                mode="single"
                selected={pendingDate ? toBoundaryDate(pendingDate) : undefined}
                defaultMonth={pendingDate ? toBoundaryDate(pendingDate) : undefined}
                onSelect={(d) => d && setPendingDate(toDate(d))}
              />

              <div className="flex items-center gap-2 px-1">
                <span className="text-muted text-xs">Time</span>

                <input
                  type="time"
                  value={pendingTime}
                  onChange={(e) => setPendingTime(e.target.value)}
                  className="border-muted-200 rounded-md border px-2 py-1 text-sm"
                />
              </div>

              <Button
                size="sm"
                onClick={() => {
                  if (pendingDate) setDateTime(`${pendingDate}T${pendingTime}:00`);
                  close();
                }}
              >
                Apply
              </Button>
            </div>
          )}
        </CalendarPicker.Content>
      </CalendarPicker>

      <span className="text-muted text-sm">{dateTime ?? 'Not set'}</span>
    </div>
  );
};

<DateTimePickerDemo />
