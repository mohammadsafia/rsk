## Variant — Neutral

File shape with no overlay. Status pill / approval prompt only render when consumer mounts them inside `Preview`. Status combobox and assignee picker are interactive — try selecting different options.

<NeutralDemo />

## Variant — Approved

`<FileCard.StatusPill variant="success">` mounted inside the file shape — content sits in front of the watermark icon.

<ApprovedDemo />

## Variant — Rejected

`<FileCard.StatusPill variant="destructive">`.

<RejectedDemo />

## Variant — Approval Required (dual-action)

`<FileCard.ApprovalPrompt>` + `<FileCard.Approve>` / `<FileCard.Reject>` as ConfirmDialog triggers. Each button is `forwardRef` so it works with `Dialog.Trigger asChild`.

<ApprovalRequiredDemo />

## Variant — Folder Mode

Mount `<FileCard.FolderShape count={...} />` inside Preview instead of `<FileCard.FileShape>`. The same card chrome (header / footer) reads as a folder. Pair with `onClick` to make the whole card a navigation target. The count comes from `accessors`-aligned entry data.

<FolderDemo />

## Sub-compound surface

| Sub-compound | Type | Auto-renders |
|---|---|---|
| `<FileCard.Header>` | container | — |
| `<FileCard.Title>` | reads context | file-icon chip + title text |
| `<FileCard.Actions>` | reads context, render-prop children | dropdown trigger + list |
| `<FileCard.Item>` | alias of `PrimeDropdown.Item` | already `withConditional(withAuth(…))` |
| `<FileCard.Separator>` | alias of `PrimeDropdown.Separator` | — |
| `<FileCard.Preview>` | reads context, render-prop children | preview region container |
| `<FileCard.FileShape>` | visual primitive | document watermark icon with content overlay |
| `<FileCard.FolderShape>` | visual primitive | folder shape with `count` |
| `<FileCard.StatusPill>` | alias of `Badge` | DS Badge with full variant set (default / success / destructive / warning / muted / secondary / neutral / accent) + size set |
| `<FileCard.ApprovalPrompt>` | visual primitive | text styling for approval-required copy |
| `<FileCard.Approve>` | wraps `ComposedButton` (outline-success) | `forwardRef` — works as `Dialog.Trigger asChild` |
| `<FileCard.Reject>` | wraps `ComposedButton` (outline-destructive) | `forwardRef` — works as `Dialog.Trigger asChild` |
| `<FileCard.Status>` | baked combobox | DI: `options`, `getOption*`, `value`, `onChange` (sync or async) |
| `<FileCard.Footer>` | container | — |
| `<FileCard.Assignee>` | baked combobox | DI: `options`, `getOption*`, `value`, `onChange` (sync or async) |
| `<FileCard.Date>` | reads context | formatted date via `accessors.getDate` |
