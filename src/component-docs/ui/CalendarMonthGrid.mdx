## Imports

```tsx
import { CalendarMonthGrid } from '@components/ui';
```

## Sub-Components

```
CalendarMonthGrid              — Root container. Scrollable column with chrome bg and thin scrollbar.
CalendarMonthGrid.Weekdays     — 7-column header strip for weekday labels. One Weekday child per column.
CalendarMonthGrid.Weekday      — One weekday header cell (e.g., "MON"). Just text.
CalendarMonthGrid.Days         — Grid container for all day cells. 7 columns, auto-rows.
CalendarMonthGrid.DayCell      — One day in the grid. Renders the day-number chip + a flex-column area for children (events).
CalendarMonthGrid.EventChip    — A button-shaped event preview inside a DayCell. Pass styling via className for type tones.
```

## Code

### Basic month

```tsx
<CalendarMonthGrid>
  <CalendarMonthGrid.Weekdays>
    {WEEKDAYS.map((label) => (
      <CalendarMonthGrid.Weekday key={label}>{label}</CalendarMonthGrid.Weekday>
    ))}
  </CalendarMonthGrid.Weekdays>

  <CalendarMonthGrid.Days>
    {days.map((day) => (
      <CalendarMonthGrid.DayCell
        key={day.iso}
        dayNumber={day.number}
        isCurrentMonth={day.isCurrentMonth}
        available={day.hasSlots}
        isToday={day.isToday}
      >
        {day.events.map((event) => (
          <CalendarMonthGrid.EventChip
            key={event.id}
            className={cn(eventTone(event).bg, eventTone(event).text, 'text-2xs truncate px-1.5 py-0.5 text-start')}
            onClick={() => openPopover(event)}
          >
            {formatEventLabel(event)}
          </CalendarMonthGrid.EventChip>
        ))}
      </CalendarMonthGrid.DayCell>
    ))}
  </CalendarMonthGrid.Days>
</CalendarMonthGrid>
```

### DayCell states

The cell automatically renders the day-number chip with appropriate styling based on three flags:

```tsx
{/* Today — primary-filled chip */}
<CalendarMonthGrid.DayCell dayNumber={5} isToday isCurrentMonth available />

{/* Current month, has availability — normal foreground */}
<CalendarMonthGrid.DayCell dayNumber={6} isCurrentMonth available />

{/* Current month, no availability — muted bg */}
<CalendarMonthGrid.DayCell dayNumber={7} isCurrentMonth available={false} />

{/* Out-of-month padding day — dimmed foreground */}
<CalendarMonthGrid.DayCell dayNumber={29} isCurrentMonth={false} available={false} />
```

### Overflow events

When a day has more events than fit visually, render the first N as `EventChip` children and add a custom "+N more" button below them. The primitive doesn't enforce a count — that's a consumer decision.

```tsx
<CalendarMonthGrid.DayCell {...dayProps}>
  {visibleEvents.map((event) => (
    <CalendarMonthGrid.EventChip key={event.id} {...chipProps}>
      {event.title}
    </CalendarMonthGrid.EventChip>
  ))}

  {overflow.length > 0 && (
    <Popover>
      <Popover.Trigger asChild>
        <Button variant="link" className="text-2xs h-auto p-0 font-medium">
          +{overflow.length} more
        </Button>
      </Popover.Trigger>
      <Popover.Content>
        {overflow.map((event) => (...))}
      </Popover.Content>
    </Popover>
  )}
</CalendarMonthGrid.DayCell>
```

## Notes

- **Pure rendering — no domain logic**. Consumers compute `dayNumber` / `isCurrentMonth` / `available` / `isToday` from their data and pass them in. The primitive doesn't iterate dates or know what a "month" is.
- **Sub-parts are independent** (no Context). Render them in any order; no Provider wrapper needed.
- **`DayCell` renders its own day-number chip** — children populate the events area below the chip.
- **`EventChip` is a `<button>`** — full button props (`onClick`, `disabled`, `aria-label`). Pair with `Popover.Trigger asChild` for click-to-popover behavior.
- **`isToday` carries `aria-current="date"` automatically** for screen-reader emphasis.
- **Overflow / "+N more" is consumer-handled** — the primitive renders whatever you put in. Use `EventChip` for the visible events plus your own overflow control below them.
